# Дастур барои ба кор андохтани санҷишҳо

Ғайр аз санҷишҳои дурустии натиҷаҳои баромад, системаи автосанҷӣ инчунин барномаи ту ва коди сарчашмаи онро месанҷад. Барои он ки санҷишҳо бомуваффақият гузаранд, бояд боварӣ ҳосил кунӣ, ки барномаи ту ба талаботи дар поён овардашуда мувофиқат мекунад. Барои ин, фармонҳои дар поён овардашударо маҳаллӣ иҷро кун ва санҷ, ки онҳо ягон хато намебароранд.

* **Санҷишҳои услубӣ.** Ҳангоми навиштани код бо _C_ ва _C++_ бояд ба _Google C++ Style Guide_ риоя кунӣ. Барои санҷидани он ки коди ту то чӣ андоза ба стандартҳои услуб мувофиқат мекунад, метавонӣ аз утилитаи _clang-format_ истифода барӣ. Дар папкаи

```materials/linters``` лежит файл ```.clang-format```, который содержит необходимые настройки для стилевого теста. Данный конфигурационный файл распространяет свое действие на все файлы, которые лежат с ним в директории или в директориях ниже. Поэтому чтобы данные настройки применились к твоим файлам с исходным кодом, скопируй ```.clang-format``` в папку src. \
 \
 Барои ба кор андохтани санҷиши услуб, фармони зеринро иҷро кун: \
 ```
clang-format -n src/sourcefile_name.c

```

 Версияи зарурии clang-format: \
 **Mac** 18.1.8 \
 **Linux** 18.1.8

 * **Статический анализ кода.** Баъзан чунин мешавад, ки барномаи C дуруст компилятсия мешавад, вале нодуруст кор мекунад ё бо хатогии кӯшиши дастрасӣ ба хотираи бегона қатъ мешавад. Барои пешгирӣ кардани чунин хатоҳо ҳангоми навиштани барнома, аз утилитаҳои махсус истифода мебаранд, ки коди сарчашмаро барои хатогиҳои эҳтимолӣ таҳлил мекунанд. Системаи автотестҳои мо барои ин ```
cppcheck

```-ро истифода мебарад.

 Ту метавонӣ коди сарчашмаи худро санҷӣ: \
 ```
cppcheck --enable=all --suppress=missingIncludeSystem src/soursefile_name.c

``` \
 Ҳамчунин метавонӣ якбора ҳамаи файлҳои коди сарчашмаро дар директория санҷӣ: \
 ```
cppcheck --enable=all --suppress=missingIncludeSystem src/```

 Версияи зарурии cppcheck: \
 **Mac** 2.13 \
 **Linux** 2.13
