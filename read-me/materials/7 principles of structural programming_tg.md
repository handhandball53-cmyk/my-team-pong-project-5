# Принсипҳои барномасозии сохторӣ

Дар асоси равиши сохторӣ ба барномасозӣ ду принсипи асосӣ меистанд:
- принсипи «тақсим кун ва ҳукмронӣ намо» (декомпозиция) — принсипи ҳалли масъалаҳои мураккаб тавассути ҷудо кардани онҳо ба шумораи зиёди вазифаҳои хурдтар ва мустақил, ки фаҳмидан ва ҳал карданашон осонтар аст. Ҳангоми ҳалли вазифаҳои хурд такрори код пешгирӣ мешавад ва худи роҳҳалҳои онҳо дар ҳолати зарурӣ дубора истифода мешаванд.
- принсипи тартиббандии иерархӣ — принсипи сохтани қисмҳои таркибии масъала дар сохторҳои иерархии дарахтмонанд бо илова кардани ҷузъиёти нав дар ҳар сатҳ (аз сатҳи болоӣ бо як нуқтаи вуруд то сатҳҳои поёнӣ бо сохторҳои мушаххаси додаҳо ва татбиқҳо). Яъне дар як сатҳ ҳам ҳисоббарорӣ ва ҳам амалиётҳои вуруд-барориш набояд ҷой дошта бошанд. Ҳамин тавр, ҳангоми истифодаи услуби сохторӣ барнома ба шакли «пироги қабат-қабат» аз боло ба поён сохта мешавад. Хатогиҳо дар сатҳҳои поёнӣ ба вуҷуд меоянд ва ба сатҳи болоӣ интиқол дода мешаванд, ки дар он ҷо ба корбар намоиш дода мешаванд.

Ташаккул ва рушди барномасозии сохторӣ бо номи Эдсгер Дейкстра вобаста аст.
* Принсипи 1. Аз истифодаи оператори гузариши бечунучаро goto даст кашидан лозим аст.
* Принсипи 2. Ҳар як барнома аз се сохтори асосии идоракунӣ сохта мешавад: пайдарпайӣ, шохабандӣ, давр.
* Принсипи 3. Дар барнома сохторҳои асосии идоракунӣ метавонанд озодона ба ҳамдигар дарунӣ шаванд. Ҳеҷ гуна воситаҳои дигари идоракунии пайдарпайии иҷрои амалиёт пешбинӣ намешавад.
* Принсипи 4. Порчаҳои такроршавандаи барномаро метавон ба шакли зербарномаҳо (процедураҳо ва функсияҳо) тартиб дод. Ба ҳамин тарз (ба шакли зербарномаҳо) порчаҳои мантиқан яклухти барномаро ҳам метавон тартиб дод, ҳатто агар онҳо такрор нашаванд.
* Принсипи 5. Ҳар як гурӯҳи мантиқан анҷомёфтаи дастурҳоро бояд ҳамчун блок тартиб дод. Блокҳо асоси барномасозии сохторӣ мебошанд.
* Принсипи 6. Ҳамаи сохторҳои зикршуда бояд як вуруд ва як баромад дошта бошанд.
* Принсипи 7. Таҳияи барнома қадам ба қадам, бо усули «аз боло ба поён» (top-down method) анҷом дода мешавад.

Оқибатҳо ва иловаҳои принсипҳои дар боло зикршуда:

1. Манъи истифодаи тағйирёбандаҳои глобалӣ. Тағйирёбандаи глобалӣ ҳар гуна тағйирёбандае ҳисобида мешавад, ки берун аз тани функсия эълон шудааст.

```cpp
int x1; // Запрещено
static int x2; // Запрещено

int sum(int a, int b) {
 int result = a + b; // Разрешено
 return result;
}
```

2. На бештар аз як баромад аз функсия. Истисно санҷиши пешакии аргументҳои функсия дар оғози тани функсия мебошад. Аммо агар имкон бошад, беҳтар аст аз ин санҷиш ҳам худдорӣ кунӣ.

```cpp
int count_negative(int *arr, int n) {
 if (arr == NULL) return -1; // Хорошо, таким образом сигнализируем об ошибке
 int count = 0;
 for (int i = 0; i < n; i++)
 if (arr[i] < 0) count++;
 return count; // Хорошо, один выход
}

int min(int a, int b) {
 if (a < b) return a; // Плохо, должен быть только один выход
 return b;
}

int max(int a, int b) {
 int result = b;
 if (a > b) result = a;
 return result; // Хорошо, так как это единственный выход
}

int factorial(int n) {
 int result = 1;
 for (int i = 1; i <= n; i++) result *= i;
 if (n < 0) return -1; // Плохо, проверка аргументов должна быть строго в начале тела функции
 return result;
}
```

3. На бештар аз як баромад аз давр — ин метавонад ҳам шарти худи давр бошад ва ҳам калимаи калидии break дар ҳолати даври беохир. Ҳамчунин истифодаи калимаи калидии `continue` манъ аст, зеро рафтори он ба оператори `goto` монанд аст.

```cpp
#define ARRAY_SIZE 5

int main(void) {
 int pos_zero = -1;
 int arr[ARRAY_SIZE] = {1, 2, 0, 4, 5};
 for (int i = 0; i < ARRAY_SIZE; i++) // Первая точка выхода
 if (arr[i] == 0) {
 pos_zero = i;
 break; // Плохо, вторая точка выхода из цикла
 }

 for (int i = 0; i < ARRAY_SIZE && pos_zero == -1; i++) // Хорошо, единственная точка выхода
 if (arr[i] == 0) pos_zero = i;

 int i = 0;
 while (true) {
 if (i == ARRAY_SIZE || pos_zero != -1) break; // Хорошо, так как это единственная точка выхода
 if (arr[i] == 0) pos_zero = i;
 i++;
 }

 for (int i = 0; i < ARRAY_SIZE && pos_zero == -1; i++) // Единственная точка выхода
 {
 if (arr[i] != 0) continue; // Запрещено
 pos_zero = i;
 }

 return 0;
}
```

4. Дарунитобии ҳар гуна блокҳо набояд аз 4 зиёд бошад. Ҳисоби дарунитобӣ аз худи функсия оғоз намешавад.

```cpp
#include <stdio.h>

#define MATRIX_COLS 3
#define MATRIX_ROWS 3

int main(void) {
 int matrix[MATRIX_ROWS][MATRIX_COLS];
 for (int i = 0; i < MATRIX_ROWS; i++)
 for (int j = 0; j < MATRIX_COLS; j++) {
 int element = matrix[i][j];

 // Подсчитываем, сколько раз element встречается в матрице
 // Лучше вынести в отдельную функцию, чтобы минимизировать уровень вложенности
 int count = 0;
 for (int k = 0; k < MATRIX_ROWS; k++)
 for (int l = 0; l < MATRIX_COLS; l++)
 if (element = matrix[k][l]) // Плохо, 5-ый уровень вложенности
 count++; // Плохо, 6-ый уровень вложенности
 }
 return 0;
}
```

6. Истифодаи функсияи анҷоми фаврии барнома exit() манъ аст.

```cpp
int sum_array(const int *arr, int n) {
 if (arr == NULL)
 exit(1); // Плохо, лучше возвращать код ошибки и обрабатывать её извне, чем прерывать программу
 int result = 0;
 for (int i = 0; i < n; i++) result += arr[i];
 return result;
}
```
